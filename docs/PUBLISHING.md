# Publishing Safely

Helios generates local build state and benchmark artifacts that include machine-specific details such as:
- absolute filesystem paths
- hostnames
- command lines with local directories
- cache and manifest locations

## Do Not Commit These
- `build/`
- `results/`
- `data/raw/`
- `data/processed/cache/`
- `data/processed/manifests/`

These are ignored by the repository `.gitignore`.

## Why You Saw CMake Files
Files such as:
- `build/CMakeCache.txt`
- `build/CMakeFiles/CMakeConfigureLog.yaml`

are generated by CMake on your machine. They are not source files and should never be pushed publicly.

## Public Artifact Flow
If you want to share proof artifacts without leaking local details:

```bash
python3 scripts/export_public_bundle.py --repo-root . --output-dir public_artifacts
```

That bundle redacts:
- `host_hostname`
- `host_nodename`

and rewrites:
- repo-local absolute paths to `$REPO_ROOT`
- home-directory paths to `$HOME`
- temp paths to `[temp-path]`

## If You Already Tracked Private Files
If you initialize git and accidentally track generated files, remove them from the index before pushing:

```bash
git rm --cached -r build results data/raw data/processed/cache data/processed/manifests
```

Then commit the `.gitignore` and only publish source, docs, and sanitized public artifacts.
